# Contributing Guidelines

Thank you for your interest. This repository is licensed for non-commercial evaluation. Contributions are subject to approval and may require a CLA.

## Ways to Contribute
- Bug reports and security reports
- Documentation improvements
- Small fixes (no new features without prior discussion)

## Workflow
1. Create an issue describing changes
2. Wait for maintainer approval
3. Fork and create a feature branch
4. Ensure lint/tests pass
5. Submit a pull request

## Legal
- By submitting a contribution, you agree it is your original work and you have the right to license it.
- You agree to assign or license contributions to Sitaram Technologies Pvt. Ltd. under the project license.
- A Contributor License Agreement (CLA) may be required before merge.

## Developer Setup
- Node/PNPM or NPM for JS projects; Python 3.10+ for Python parts
- Follow READMEs in `backend/` and `frontend/`

## Coding Standards
- Follow existing style and `eslint`/`tsc` or `flake8`/`mypy`
- Add tests for non-trivial changes
